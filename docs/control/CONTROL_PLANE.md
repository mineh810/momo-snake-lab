# Control Plane

The control plane owns product intent, safety boundaries, task contracts, CI/CD gates, review expectations, and release criteria.

## Assets

- PRD: `docs/product/PRD.md`
- Taste: `docs/product/TASTE.md`
- Specs: `docs/specs/`
- Review guide: `docs/ops/REVIEW_GUIDE.md`
- GitHub Actions: `.github/workflows/`
- PR template: `.github/PULL_REQUEST_TEMPLATE.md`

## Responsibilities

- Define what should be built
- Define what must not be built
- Keep work issue/spec driven
- Require tests for game logic
- Require screenshots or preview notes for UI changes
- Keep deployment GitHub-native
